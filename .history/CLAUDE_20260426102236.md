# August Digital — Frontend Rules (Next.js)

## 🎨 Branding & Style
- **Brand Name:** August Digital
- **Design Concept:** Minimalist, Futuristic, Tech-focused (Water element accents).
- **UI Framework:** Tailwind CSS + Shadcn/UI.
- **Iconography:** Lucide React hoặc Heroicons.
- **Typography:** Ưu tiên font Sans-serif hiện đại, sạch sẽ.

## 🛠 Tech Stack
- **Framework:** Next.js 14+ (App Router).
- **Language:** TypeScript (Strict mode).
- **Data Fetching:** SWR hoặc TanStack Query (React Query) để quản lý cache và polling cho Notification.
- **State Management:** Zustand (để quản lý giỏ hàng và thông tin User).

## 📡 API Guidelines
- **Base URL:** Lấy từ `process.env.NEXT_PUBLIC_API_URL`.
- **Naming Convention:** - Backend trả về `snake_case`. 
  - Frontend convert sang `camelCase` khi mapping vào Interface (hoặc giữ nguyên nếu muốn đồng nhất nhanh).
- **Authentication:** Gửi JWT qua Header `Authorization: Bearer <token>`.
- **Error Handling:** Sử dụng `ApiResponse<T>` chuẩn từ Backend để hiển thị Toast thông báo lỗi.

## 🏗 Coding Standards
- **Components:** Ưu tiên Functional Components, sử dụng Server Components cho SEO và Client Components (`'use client'`) cho tương tác.
- **File Naming:** - Thư mục: `kebab-case`.
  - Component file: `PascalCase.tsx`.
  - Service/Hook file: `camelCase.ts`.
- **Props:** Luôn định nghĩa Interface/Type rõ ràng cho Props.

## 🔄 Workflow Rules
- Luôn kiểm tra quyền truy cập (Role-based access) cho các route `/admin`.
- Notifications: Implement polling (30s/lần) hoặc cơ chế làm mới badge khi người dùng chuyển trang.
- Warranty: Luôn hiển thị trạng thái (Badge) rõ ràng: OPEN, RESOLVED, PENDING_STOCK.
